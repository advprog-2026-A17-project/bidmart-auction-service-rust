# API — Modul 3 (Lelang & Bidding)

Gateway: `/api/v1/listings/**` → auction-service:8082

OpenAPI: [openapi.yaml](./openapi.yaml)

`{listingId}` = ID listing katalog (modul 2). Layanan ini **tidak** menyimpan data produk katalog.

## Alur

1. Seller: publish listing (modul 2) → `POST /api/v1/listings` buka sesi lelang
2. Buyer: `POST /api/v1/listings/{listingId}/bids` (hold modul 4)
3. Katalog terima event harga terbaru
4. Tutup: scheduler atau `POST .../close` → WON/UNSOLD

## Endpoint

| Method | Path | Keterangan |
|--------|------|------------|
| GET | `/api/v1/listings` | Daftar sesi |
| GET | `/api/v1/listings/{listingId}` | Detail sesi |
| POST | `/api/v1/listings` | Buka sesi (idempotent) |
| POST | `/api/v1/listings/{listingId}/bids` | Place bid |
| GET | `/api/v1/listings/{listingId}/bids` | Riwayat bid |
| GET | `.../bids/cursor` | Paginasi cursor |
| POST | `.../bids/cursor` | Proxy/max bid |
| POST | `/api/v1/listings/{listingId}/close` | Settlement |
| GET | `/api/v1/listings/pending-closure` | Untuk scheduler |

## Header

- `X-User-Id` dari gateway (identitas bidder/seller)
- Konflik `bidderId`/`sellerId` di body → 403

## Metrics

`GET /metrics` — Prometheus (modul 8)
